Migliorare e mantenere
noiza-pdf-skill

Per: chi vuole aggiungere componenti, correggere bug o modificare il workflow
Prerequisito: accesso in write al repo NoizaDev/noiza-pdf-skill

1. Struttura del repo

noiza-pdf-skill/
├── SKILL.md              ← istruzioni per Claude (il "cervello" della skill)
├── README.md             ← guida installazione e uso rapido
├── guide-utilizzo.md     ← guida per chi genera documenti
├── guide-contribuzione.md← questa guida
└── references/
    ├── template.html     ← CSS Noiza completo (unica fonte di stile)
    └── components.md     ← catalogo componenti con snippet HTML

2. Cosa NON modificare senza test

SKILL.md — controlla il comportamento di Claude. Una modifica sbagliata cambia come la skill interpreta le richieste per tutti.

CSS in template.html — qualsiasi modifica si riflette su tutti i documenti rigenerati. Testa sempre su un documento reale prima di fare push.

3. Aggiungere un nuovo componente

Un componente è una classe CSS riutilizzabile. Segui questi 3 passi nell'ordine:

Passo 1 — CSS in references/template.html

Aggiungi il nuovo stile nella sezione commenti corrispondente:

.nome-componente {
  background-color: #f0fdf4;
  border-left: 3px solid #16a34a;
  padding: 10px 14px;
  margin: 10pt 0;
}
.nome-componente p, .nome-componente li {
  font-size: 11pt;
  padding-top: 3pt;
  padding-bottom: 3pt;
}

Passo 2 — Esempio in references/components.md

Aggiungi una sezione con: titolo, descrizione breve, snippet HTML pronto all'uso. Segui il formato degli altri componenti nel file.

Passo 3 — Tabella in SKILL.md

Aggiungi una riga alla tabella "Componenti disponibili" con: nome, classe CSS, quando usarlo.

5. Fare una PR

  1. Crea un branch con nome descrittivo: add-success-box, fix-table-overflow
  2. Fai le modifiche nei 3 file coinvolti (CSS, components.md, SKILL.md)
  3. Testa generando almeno un PDF completo con il nuovo componente
  4. Apri la PR su main con una riga di descrizione nel titolo

Commit message consigliato:

feat: aggiungi componente .success-box (verde)
fix: correggi overflow tabelle con testo lungo

Regola base: i file .md sono la sorgente, i file .html e .pdf sono output rigenerabili. Nel repo versiona solo i .md e i file di configurazione — non i PDF generati.
